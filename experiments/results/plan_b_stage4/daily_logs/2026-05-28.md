## Decisions made

- Treat the old 1000 high-joint labels only as dev / threshold diagnostic data.
- Treat the completed 3000 score-space stratified labels as the final held-out Stage 4 annotation set for the ICDM revision.
- Do not tune thresholds on the 3000 held-out labels; report fixed dev-selected thresholds for the main evaluation.
- Record the annotator's score-assisted label rule: if both `image_similarity` and `text_similarity` are `>0.85` and `<0.95`, `near-duplicate` is acceptable; if both are `>0.95`, `duplicate` is acceptable. This is annotation guidance, not an evaluation-threshold tuning step.
- For downstream v2 split generation, use `B=image>=0.85`, `C=text>=0.95`, `D=image>=0.85 OR text>=0.95`, and `E=image>=0.85 AND text>=0.85` with `strict_duplicate_if_both>=0.95`.
- Treat old LLaVA training and VQAv2 quick evaluation as diagnostic only; do not use them as final ICDM downstream claims.
- Use periodic LoRA adapter checkpoints (`checkpoint-step-*`) and `--resume` for v2 downstream training to reduce risk from Windows/WSL interruptions.
- Launch long v2 downstream jobs via Windows Task Scheduler rather than a plain SSH-launched background WSL process, because plain WSL background/tmux sessions did not survive SSH disconnect in testing.
